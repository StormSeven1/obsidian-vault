# 一套通用中文风格前缀

建议每张图前面都先拼这一段：

企业级系统架构信息图，中文科技架构海报，矢量风格，画面干净克制，层次分明，结构清晰，现代高端 B 端产品设计风格，白底或深蓝底，蓝灰色发光连线，模块化信息布局，阅读性强，专业、理性、可信，适合汇报 PPT 首页，避免花哨装饰，避免赛博朋克，避免科幻城市，避免机器人大脑，避免无意义电路纹理，强调治理、边界、流程、状态真相链

通用负面约束也可以加：

不要拥挤排版，不要过多小字，不要乱码，不要难以辨认的中文，不要复杂背景，不要卡通，不要 3D 玩具风，不要过强霓虹，不要多个中心大脑，不要混乱箭头，不要无意义图标堆砌

# 1）中文总架构图提示词

这张图用于先讲清楚“整个系统是什么”。

## 提示词

企业级系统架构信息图，中文科技架构海报，矢量风格，白色背景，结构清晰，模块层级鲜明，蓝灰色连线，专业克制，适合产品架构汇报  
  
请绘制“态势感知 AI-native Agent 系统总体架构图”。  
  
核心要求：  
整个系统必须表现为“单一控制平面”，不是多个并列 Agent，不是多个主脑。  
画面中央是“控制内核”，作为唯一治理权威。  
围绕控制内核分层展示 7 个运行平面：  
  
1. 底部为“世界接口平面”：连接外部世界、用户、传感器、雷达、视频、GIS、数据库、资产设备  
2. 世界接口上方为“运行边界平面”：显示五个关卡——准入关卡、执行关卡、资产控制关卡、提交关卡、人工接管关卡  
3. 控制内核左侧为“审议与计划平面”  
4. 控制内核右侧为“执行平面”  
5. 控制内核下方为“状态与知识平面”，体现运行时状态与长期记忆  
6. 顶部为“操作员与可观测平面”，体现审批、接管、回放、评估  
7. 右下角或侧边单独画出“Consolidation Run / 整合运行”，作为独立后台演进回路，与实时链路明显分开  
  
图中重点表达：  
- 单一控制平面  
- 边界先于规划  
- 执行层不能直接写入最终真相  
- 状态与知识平面是系统真相基础  
- 人始终在环  
- 学习演进与实时关键路径彻底分离  
  
中文标签要大而清晰，整体像高端产品架构总览图，简洁、可信、易懂

# 2）中文功能组成图提示词

这张图比总架构图更偏“模块职责说明”。

## 提示词

中文企业级功能架构图，矢量信息图，深蓝背景，科技感但克制，结构化模块布局，标签清晰，适合 AI 平台解决方案介绍  
  
请绘制“态势感知 AI-native Agent 系统功能组成图”。  
  
把系统拆成多个功能模块，每个模块是一个大框，采用清晰网格布局，突出职责分离。主要模块包括：  
  
- 世界接口平面：协议适配、数据标准化、复杂事件处理、指令回执  
- 运行边界平面：权限边界、区域与时间窗约束、风险约束、审批停点、回滚前提  
- 控制内核：事件接入、运行准入、生命周期管理、调度、仲裁、提交审查、降级恢复  
- 审议与计划平面：上下文装载、证据组织、态势研判、计划生成、任务重规划  
- 执行平面：工作流、技能、工作单元、直接动作  
- 状态与知识平面：观测事实、证据图、判断与假设、工作态、已提交状态、长期记忆  
- 操作员与可观测平面：审批、接管、停止、恢复、追溯、回放、诊断、评估  
- 整合运行：从归档中蒸馏记忆、提出新技能候选、提出工作流候选、提出策略优化建议  
  
每个模块下面只保留 3 到 5 个简洁中文关键词。  
整体观感要像高级产品白皮书中的功能架构页，不要画成代码类图，不要过多技术细节文字。

# 3）中文主循环流程图提示词

这张图用来解释系统怎么运行。

## 提示词

中文流程架构图，企业级运行流程信息图，白底，简洁箭头，强结构顺序感，适合技术汇报  
  
请绘制“态势感知 AI-native Agent 系统 10 步运行主循环流程图”。  
  
按照严格线性顺序绘制以下 10 个步骤：  
1. 事件接入  
2. 边界检查  
3. 运行决策  
4. 上下文装载  
5. 计划形成  
6. 执行  
7. 验证  
8. 提交与发布  
9. 监控与重入  
10. 归档  
  
要求重点突出：  
- “边界检查”必须在“计划形成”之前  
- “验证”必须在“提交与发布”之前  
- “提交与发布”由控制内核控制  
- 快速运行可以跳过部分深度审议，但不能跳过关键治理环节  
- 学习与演进不在这个主循环里  
  
可在每个步骤旁边用小字标明所属平面：  
世界接口、运行边界、控制内核、审议与计划、执行、验证、状态与知识  
  
在主流程旁边额外画一个分离的小回路“整合运行 / Consolidation Run”，由“归档”流出，但与主循环分开，表示后台演进。  
整体效果要严谨、清爽、一眼能看懂。

# 4）中文双环图提示词：实时链路 vs 后台演进

这张图很适合讲“为什么这架构优雅”。

## 提示词

中文双回路系统架构图，现代企业级信息图，白底，留白充足，理性克制，强调系统思维  
  
请绘制“实时任务主链路 与 后台整合演进回路”的双环架构图。  
  
左侧是“实时任务主链路”：  
- 事件驱动  
- 时延敏感  
- 可审计  
- 可被操作员中断与接管  
- 由运行边界和控制内核治理  
  
右侧是“后台整合演进回路”：  
- 定时触发  
- 读取任务归档、证据、追溯记录、长期记忆  
- 蒸馏经验与记忆  
- 产出新技能候选、新工作流候选、策略优化建议  
- 不能写入已提交状态  
- 不能直接上线生产能力  
- 必须经过人工复核  
  
两个回路中间通过“归档池 / Trace Pool”和“能力提议队列”连接。  
要明显表现：  
“系统会持续学习变好，但不会污染实时控制主链路”  
  
整体要像架构思想图，而不是普通泳道图。

# 5）中文真相链图提示词

这张图非常适合帮助用户理解“为什么系统可信”。

## 提示词

中文概念型架构信息图，深色背景，蓝色科技线条，清晰层级堆叠，专业可信，适合讲解 AI 系统状态治理  
  
请绘制“系统单一真相链”示意图，采用自下而上或自左向右的层级推进结构，展示五层状态链：  
  
1. 观测事实  
2. 证据图  
3. 判断 / 假设 / 推演预测  
4. 工作态  
5. 已提交状态  
  
要求表达：  
- 事实不等于判断  
- 判断不等于最终真相  
- 工作草稿不等于全局生效状态  
- 推演预测是基于证据的未来判断  
- 只有控制内核通过提交关卡，才能把结果升级为“已提交状态”  
  
在旁边适当连接“验证”“审查记录”“操作员监督”。  
最终“已提交状态”要表现得稳定、加固、经过治理审查。  
不要画成数据库表结构，要画成概念清晰、治理严谨的状态真相推进图。

# 6）中文控制内核拆解图提示词

这张图能帮助你介绍“唯一治理权威，但不是臃肿大总控”。

## 提示词

中文控制中枢分解架构图，企业级技术海报，白底，中央主模块 + 五个职责分区，简洁专业，适合高层方案讲解  
  
请绘制“控制内核分解图”。  
  
控制内核必须是系统唯一治理权威，位于画面中心。  
但其内部拆分为五个职责模块：  
  
1. 事件接入与运行准入  
2. 生命周期与调度  
3. 策略与审查协调  
4. 提交与审计权威  
5. 韧性与降级控制  
  
图中要强调：  
- 只有一个治理中心  
- 但不是一个巨型上帝类  
- 控制内核不负责复杂业务推理  
- 控制内核不负责领域分析  
- 它负责治理、仲裁、生命周期与最终提交  
  
将控制内核与以下外围平面相连：  
运行边界、审议与计划、执行平面、状态与知识、操作员与可观测平面  
  
整体效果要成熟、克制、精密，避免夸张 AI 大脑造型。

# 7）中文四个正交维度图提示词

这张图很适合讲清楚“不是按业务场景一层层堆 Agent”。

## 提示词

中文概念矩阵架构图，白底，信息设计高级感，逻辑清晰，适合架构讲解  
  
请绘制“系统运行模式的四个正交维度图”。  
  
通过四个维度展示系统运行方式的组合关系，而不是画多个并列业务 Agent。四个维度为：  
  
- 触发维度：用户触发、外部事件触发、定时触发、回调触发、策略触发、记忆触发  
- 认知深度维度：快速运行、深度审议运行、任务持续运行  
- 治理维度：观察建议、半自动、审批后继续、人工控制  
- 执行载体维度：直接动作、工作流、技能、工作单元  
  
图中要表达：  
- 这四个维度彼此正交  
- 任一次运行都是四个维度的组合  
- 新需求应该落入现有维度组合，而不是再长出新的系统层  
- 架构不会随着场景增加而裂变成多个主脑  
  
整体像一张很清晰的架构教育图，不要画成数学坐标图。

# 8）中文五个 Gate 治理图提示词

这张图适合强调“可控、安全、可审计”。

## 提示词

中文治理与风控架构图，深蓝背景，企业级高可信视觉风格，清晰拦截点和关卡结构，理性、严谨、强治理感  
  
请绘制“系统五大治理关卡图”。  
  
展示五个关键关卡，以及它们在系统运行中的拦截位置：  
  
- 准入关卡：在事件进入系统、创建运行实例之前  
- 执行关卡：在计划步骤执行之前  
- 资产控制关卡：在资产租约获取和动作下发之前  
- 提交关卡：在判断升级为已提交状态之前  
- 人工接管关卡：在人工接管、停止、回滚、恢复时进入统一治理链  
  
重点表达：  
- 所有关键动作都必须命中明确关卡  
- 关卡是确定性治理机制，不依赖模型临场判断  
- 关卡与审批、风险控制、审查记录、回滚前提紧密相连  
- 系统的关键能力来自“明确边界 + 统一治理”  
  
整体像风险控制架构图，不要做成抽象艺术图。

# 更适合中文图生成的简化版策略

如果你要提高中文出图成功率，我建议把提示词控制成：

- **主标题**
- **6 到 10 个大标签**
- **少量关键词**
- **明确布局要求**
