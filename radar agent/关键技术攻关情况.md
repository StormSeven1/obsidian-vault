# 三阶分层记忆架构关键技术攻关情况

三阶分层记忆架构是雷达智能体从研究内容走向工程实施的关键技术主线。该架构面向智能雷达调度、波形参数设置和当前环境目标研判三类核心需求，支撑智能体在任务执行过程中保持上下文，在近期任务中形成连续协同，在长期运行中沉淀雷达专业知识、参数策略和处置经验。

项目围绕“短期任务记忆、中期任务连续记忆、长期知识经验记忆、层间流转与经验治理”四个方向开展技术攻关，形成从当前任务支撑到经验复用的完整技术闭环。

## 1. 短期记忆层关键技术

短期记忆层面向单次雷达智能体任务，主要支撑目标研判、波形参数优化、工具调用、工作流执行和显控系统交互中的上下文一致性。它在一次任务生命周期内保存当前目标、区域、模型输出、证据、候选假设、候选参数方案、工具调用结果和工作流检查点。

当前攻关重点包括以下内容。

1. 雷达任务工作空间建模。围绕目标、区域、时间窗口、任务目标、参数方案、证据链和执行状态建立统一任务工作空间，使智能体在一次复杂任务中拥有稳定认知视角。
2. 端到端模型输出摘要。将目标检测、识别、跟踪、异常评分和置信度等模型输出转化为智能体可检索、可推理、可展示的结构化证据。
3. 工具调用中间结果管理。保存每次能力调用的输入、输出、异常、耗时和结果摘要，支撑多轮推理中的观察、修正和复盘。
4. 候选参数方案管理。对波形参数、检测门限、跟踪参数和资源调度方案进行结构化保存，记录推荐依据、适用范围、预期收益、风险点和回滚方案。
5. 运行检查点与恢复。为工作流节点、参数调整前状态、工具调用结果和智能体推理状态建立检查点，支撑异常后的恢复、回放和复盘。
6. 上下文预算控制。根据任务类型选择进入模型上下文的关键信息，保证复杂任务中证据充分、上下文紧凑、调用成本可控。

通过短期记忆层攻关，智能体能够在一次目标研判或参数优化任务中持续跟踪上下文，保持多轮推理、多工具调用和工作流协同的一致性。

## 2. 中期记忆层关键技术

中期记忆层面向近期任务链和人机协同，主要支撑显控系统中的跨轮次追问、连续目标分析、参数调整效果对比、操作员偏好复用和未完成任务管理。它将多次相关任务组织成目标、区域或参数维度的连续任务线索。

当前攻关重点包括以下内容。

1. 跨会话实体指代解析。支持“刚才那个目标”“前一次参数方案”“A 区域上次异常”等显控交互表达，建立用户语言、界面上下文和历史任务对象之间的关联。
2. 任务链路关联。围绕同一目标、同一区域、同一异常事件或同一参数调整目标聚合近期任务，形成可追踪任务线。
3. 目标研判时间线。记录目标状态、航迹质量、模型置信度、异常评分、研判结论和证据变化，支撑连续目标分析。
4. 参数调整历史回看。保存参数调整前状态、候选方案、审批记录、执行结果、调整后指标变化和操作员反馈，支撑后续参数策略优化。
5. 工作流执行历史管理。记录异常目标研判、波形参数调整、跟踪质量评估和回滚恢复等工作流的运行过程和节点结果。
6. 操作员偏好建模。学习操作员关注的证据类型、报告粒度、参数确认习惯、常用工作流和界面交互方式，使智能体输出更贴合显控使用场景。

通过中期记忆层攻关，智能体能够从单次问答式交互升级为连续任务协同，支持操作员围绕目标、区域和参数状态进行持续追问、对比和调整。

## 3. 长期记忆层关键技术

长期记忆层面向雷达专业知识和历史经验沉淀，主要支撑目标研判、异常归因、波形策略推荐、相似案例检索、工作流优化和能力路由。它将雷达知识、历史案例和运行经验组织为可检索、可验证、可复用的知识资产。

当前攻关重点包括以下内容。

1. 雷达技术文档向量化。将设备手册、操作规程、波形说明、算法说明和故障处置文档转化为语义向量，支撑自然语言检索和相似知识召回。
2. 雷达领域知识图谱构建。围绕目标类型、环境因素、雷达参数、设备状态、干扰样式、故障模式和处置策略建立关系网络，支撑异常归因和策略推荐。
3. 相似案例检索。基于目标特征、航迹变化、模型置信度、环境状态、参数配置和异常表现召回历史案例，为当前研判和参数方案生成提供依据。
4. 成功与失败参数案例抽取。记录不同任务、环境和目标条件下参数调整的效果，形成波形策略、门限策略、跟踪策略和回滚策略的经验库。
5. 异常处置模式标签生成。对历史任务中的异常原因、有效证据、处置动作和最终效果进行抽象，形成可复用模式标签。
6. 知识冲突检测。识别新经验与既有知识、参数策略、工作流模板之间的差异，支撑专家复核和版本治理。
7. 专家验证与发布。将候选经验资产经过评估、复核和发布流程后，回流到能力检索、参数策略推荐和工作流模板优化中。

通过长期记忆层攻关，智能体能够在长期运行中逐步形成雷达领域知识库、典型场景案例库、参数策略库和经验资产库，为复杂目标研判和智能调度提供稳定支撑。

## 4. 层间流转与经验治理关键技术

三阶分层记忆的工程价值来自层间流转。项目采用“任务追溯记录驱动经验沉淀”的技术路线，将短期运行过程、中期任务链和长期经验资产串联起来。

经验流转流程如下。

```text
短期任务运行
  → 追溯记录 Trace
  → 中期任务链聚合
  → 候选经验片段
  → 情景摘要与模式标签
  → 评估样例与影子运行
  → 已发布经验资产
  → 能力检索、参数策略和工作流模板回流
```

追溯记录保存任务输入、上下文快照、端到端模型输出、能力调用、工作流节点、候选判断、参数方案、审批确认、执行结果、效果指标和操作员反馈。系统从追溯记录中抽取候选经验片段，再结合中期任务链进行聚合，形成情景摘要、模式标签和评估样例。

经验治理机制围绕来源可追溯、状态可管理、效果可评估、影响可控制四个方面展开。每个经验资产记录来源任务、证据引用、适用条件、验证状态、发布状态、影响范围和版本信息。已发布经验资产参与相似案例检索、能力排序、参数策略推荐和工作流优化；候选经验资产进入评估和影子运行流程，为后续发布提供依据。

通过层间流转与经验治理攻关，三阶分层记忆架构能够把智能体每一次目标研判、参数调整和工作流执行转化为可积累的工程资产，使系统在保持雷达运行安全边界的同时实现持续学习和能力进化。

## 5. 阶段成果形态

三阶分层记忆架构攻关形成以下阶段成果。

1. 短期记忆成果：雷达任务工作空间、模型输出摘要对象、候选参数方案对象、工具调用观察对象、运行检查点对象。
2. 中期记忆成果：目标研判时间线、参数调整历史、任务链路索引、工作流执行历史、操作员偏好画像。
3. 长期记忆成果：雷达知识库、领域知识图谱、相似案例库、参数策略库、异常处置经验库、经验资产库。
4. 治理成果：追溯记录规范、经验资产生命周期、评估样例库、影子运行机制、经验发布流程。

这些成果共同支撑雷达智能体从当前任务执行、近期任务协同到长期经验沉淀的完整能力闭环，为后续智能雷达调度、波形参数优化和当前环境目标研判的工程落地提供关键技术基础。
