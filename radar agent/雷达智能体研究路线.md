# 雷达智能体研究路线

## 1. 雷达智能体研究目标与设计定位

雷达智能体研究面向智能雷达系统的认知增强与人机协同需求，目标是在现有雷达感知、处理、控制和显控体系基础上，构建面向雷达任务的智能推理能力，使系统能够围绕当前环境目标研判、智能调度、波形参数设置和任务经验沉淀，形成可解释、可追溯、可演进的智能辅助决策闭环。

从建设目标看，雷达智能体不是简单接入大语言模型，也不是脱离雷达工程链路的通用问答系统，而是将大模型推理、雷达专业知识、任务上下文、专业工具、工作流和三阶分层记忆组织为统一的任务运行机制。系统需要能够理解显控指令、感知结果事件和雷达状态告警，将其转化为结构化任务；能够综合目标、环境、设备状态、历史案例和专业知识形成研判依据；能够生成目标研判结论、调度建议和波形参数候选方案；能够将参数设置、控制请求适配、人工确认和异常回滚等高风险环节纳入受控流程；能够在任务结束后沉淀过程记录、操作反馈和执行效果，支撑后续复盘与经验复用。

从系统定位看，雷达智能体属于雷达内部的智能推理与协同子系统，服务对象主要是显控分系统、信息处理感知子系统、雷达状态数据、数据知识底座和控制执行链路。其核心作用是把分散的感知结果、设备状态、历史经验和操作意图组织成面向任务的认知工作空间，并在工程约束下生成可供操作员理解、确认和使用的研判结果与候选方案。涉及实际控制下发的内容，应通过工作流、安全门控和控制请求适配机制进入受控执行链路，保证智能推理能力与雷达控制链路协同运行。

从研究边界看，本研究聚焦雷达智能体自身的任务理解、智能推理、能力调用、受控流程、记忆演进和追溯治理，不扩展为指挥所级态势感知智能体，也不替代其他雷达分系统的专业功能设计。信息处理感知子系统负责提供目标检测、识别、跟踪、定位和置信度等感知结果；控制执行相关分系统负责完成真实控制指令下发和执行反馈；雷达智能体重点完成任务建模、上下文组织、研判推理、候选方案生成、流程触发、结果解释和经验回流。

## 2. 雷达智能体关键能力体系

雷达智能体的研究路线应围绕关键能力体系展开，而不是按照软件模块、接口表或部署单元重复描述。关键能力体系体现的是建设雷达智能体必须形成的核心技术能力，以及这些能力在雷达任务闭环中的作用。

第一类是任务理解与运行组织能力。雷达智能体需要统一承接显控自然语言指令、目标选择、区域选择、告警卡片、感知结果事件和雷达状态告警，将不同来源、不同形式的输入转化为结构化任务对象。任务对象应包含任务类型、触发来源、目标或区域引用、时间窗口、风险等级、上下文版本、执行状态和追溯标识等信息。通过统一任务对象，系统能够把后续上下文装载、能力选择、推理规划、工具调用、工作流流转、结果发布和经验回流组织在同一任务主线下，避免智能体推理过程与雷达业务对象脱节。

第二类是雷达态势认知与证据组织能力。雷达智能体需要面向当前任务形成可推理、可解释、可引用的任务工作空间。该能力不是简单汇总数据，而是围绕目标研判和参数推荐需要，将信息处理感知结果、雷达工作状态、环境条件、历史任务、相似案例、专业知识、时序指标和图关系信息进行组织，区分观测事实、模型判断、历史经验、推理结论和人工反馈。系统输出目标类别、行为特征、异常程度、威胁等级、证据缺口和处置建议时，应能够给出对应证据来源和适用边界，使研判结果能够被显控展示、人工复核和后续复盘使用。

第三类是智能推理与参数推荐能力。雷达智能体需要在大语言模型推理引擎、雷达专业能力包和任务上下文共同约束下开展推理，生成面向雷达业务的结构化结果。对于当前环境目标研判任务，系统重点形成目标解释、异常归因、跟踪质量评估、证据缺口提示和处置建议；对于智能调度和波形参数设置任务，系统重点形成工作模式、波形参数、波束资源、检测门限和跟踪参数等候选方案，并说明推荐依据、适用条件、预期效果、风险点和回滚条件。该能力的核心不是让模型直接替代控制逻辑，而是让模型在专业知识和工程约束下提供可审查、可比较、可进入流程的候选决策。

第四类是工具调用与受控流程能力。雷达智能体需要把开放式推理能力和确定性工程流程结合起来。对目标解释、数据查询、状态分析、报告生成等只读研判任务，可通过推理-行动-观察式工具调用完成动态分析；对参数仿真、波形参数调整、工作模式切换、控制请求适配和异常回滚等标准化或高风险任务，应进入工作流受控流程。工作流应承接参数边界校验、仿真评估、人工确认、检查点记录、控制请求适配、状态跟踪和异常处理，使智能体生成的候选方案能够在可控、可审计的链路中推进。

第五类是记忆沉淀与持续演进能力。雷达智能体需要通过短期记忆、中期任务链和长期经验资产支撑连续任务和能力演进。短期记忆服务于当前任务，保存任务上下文、工具结果、工作流状态和中间推理依据；中期记忆服务于连续任务链，维护目标、区域、参数方案、操作反馈和任务状态之间的关联；长期记忆服务于知识、案例、策略和经验资产沉淀，保存经过评估的复盘结论和可复用经验。记忆沉淀应与追溯记录、效果评估和人工反馈结合，形成经验候选、评估确认、受控复用的闭环，避免把一次性任务记录直接作为长期经验使用。

上述五类能力共同构成雷达智能体的研究主线：任务理解与运行组织解决“任务如何进入智能体”；态势认知与证据组织解决“推理依据从哪里来”；智能推理与参数推荐解决“智能体生成什么结果”；工具调用与受控流程解决“结果如何进入工程链路”；记忆沉淀与持续演进解决“任务经验如何反馈到后续能力提升”。通过这一能力体系，雷达智能体能够从单次智能问答转向面向雷达任务的持续运行系统。

## 3. 雷达智能体技术路线与阶段成果

雷达智能体技术路线以任务闭环为主线展开，整体流程为：任务触发、任务实例化、上下文装载、LLM 与雷达 Skills 推理、工具或工作流受控调用、结果发布、追溯记录、记忆回流。该路线将大模型推理能力纳入雷达工程约束，使智能体既具备开放式研判和自然语言交互能力，又能够通过工作流、门控校验、追溯记录和记忆治理保持工程可控。

在任务触发阶段，显控指令、感知结果事件和雷达状态告警进入雷达智能体，由任务运行机制创建任务实例，绑定任务编号、目标或区域引用、任务类型、风险等级和追溯标识。在上下文装载阶段，系统围绕任务实例加载当前感知结果、雷达状态、历史任务、相似案例、专业知识和数据质量信息，形成任务工作空间。在智能推理阶段，LLM 推理引擎结合雷达 Skills 能力包完成意图理解、任务建模、执行路径选择和结果生成。对于只读研判和推荐生成任务，系统通过工具调用补充证据、分析状态和生成结果；对于参数设置和控制相关任务，系统将候选方案提交受控工作流，完成参数边界校验、仿真评估、人工确认、控制请求适配和状态反馈。任务结束后，系统将输入、上下文版本、工具结果、工作流节点、审批动作、输出结论、操作员反馈和执行效果写入追溯记录，并按照短期、中期、长期记忆机制进行归档、评估和回流。

第一阶段形成基础能力验证成果。该阶段重点完成任务建模、上下文组织、三阶记忆和追溯记录框架，验证雷达智能体能否稳定接收显控任务、感知事件和状态告警，能否形成结构化任务对象，能否围绕目标、区域、时间窗口和风险等级组织上下文，能否记录任务过程并形成可复盘的追溯链。阶段成果包括任务对象样例、上下文组织样例、三阶记忆框架、追溯记录样例和基础任务闭环原型。

第二阶段形成典型场景验证成果。该阶段重点围绕当前环境目标研判、智能调度建议和波形参数推荐开展验证，检验智能体能否综合感知结果、雷达状态、环境条件、历史案例和专业知识生成可解释结果。目标研判场景应形成目标类别、行为特征、异常程度、证据链和处置建议；调度与波形参数推荐场景应形成候选参数方案、推荐依据、适用条件、预期效果、风险说明和回滚条件。阶段成果包括典型任务样例库、目标研判演示样例、参数推荐演示样例、证据链输出模板和显控展示样例。

第三阶段形成闭环演进验证成果。该阶段重点验证工具调用、工作流受控执行、复盘评估和经验沉淀能力，推动雷达智能体从单次任务处理走向持续运行和持续优化。系统应能够将参数仿真、人工确认、控制请求适配、执行状态反馈和异常回滚纳入受控流程；能够基于任务追溯记录和执行效果生成复盘结论；能够将经过评估的任务经验沉淀为长期经验资产，并在后续相似任务中受控引用。阶段成果包括受控工作流样例、参数调整闭环演示、复盘评估报告样例、经验资产样例和持续优化验证结果。

通过上述三个阶段，雷达智能体研究路线从基础任务闭环入手，逐步扩展到典型雷达智能应用场景，再进入受控执行和经验演进闭环。最终形成面向雷达内部任务的智能推理、参数推荐、流程协同和持续学习能力，为后续软件需求规格说明、概要设计、接口控制文档和原型系统建设提供研究基础。
