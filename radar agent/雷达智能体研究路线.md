# 雷达智能体研究路线

## 1. 雷达智能体研究目标与设计定位

雷达智能体研究面向智能雷达系统的认知增强与人机协同需求，目标是在现有雷达感知、处理、控制和显控体系基础上，构建面向雷达任务的智能推理能力，使系统能够围绕当前环境目标研判、智能调度、波形参数设置和任务经验沉淀，形成可解释、可追溯、可演进的智能辅助决策闭环。

从建设目标看，雷达智能体不是简单接入大语言模型，也不是脱离雷达工程链路的通用问答系统，而是将大模型推理、雷达专业知识、任务上下文、专业工具、工作流和三阶分层记忆组织为统一的任务运行机制。系统需要能够理解显控指令、感知结果事件和雷达状态告警，将其转化为结构化任务；能够综合目标、环境、设备状态、历史案例和专业知识形成研判依据；能够生成目标研判结论、调度建议和波形参数候选方案；能够将参数设置、控制请求适配、人工确认和异常回滚等高风险环节纳入受控流程；能够在任务结束后沉淀过程记录、操作反馈和执行效果，支撑后续复盘与经验复用。

从系统定位看，雷达智能体属于雷达内部的智能推理与协同子系统，服务对象主要是显控分系统、信息处理感知子系统、雷达状态数据、数据知识底座和控制执行链路。其核心作用是把分散的感知结果、设备状态、历史经验和操作意图组织成面向任务的认知工作空间，并在工程约束下生成可供操作员理解、确认和使用的研判结果与候选方案。涉及实际控制下发的内容，应通过工作流、安全门控和控制请求适配机制进入受控执行链路，保证智能推理能力与雷达控制链路协同运行。

从研究边界看，本研究聚焦雷达智能体自身的任务理解、智能推理、能力调用、受控流程、记忆演进和追溯治理，不扩展为指挥所级态势感知智能体，也不替代其他雷达分系统的专业功能设计。信息处理感知子系统负责提供目标检测、识别、跟踪、定位和置信度等感知结果；控制执行相关分系统负责完成真实控制指令下发和执行反馈；雷达智能体重点完成任务建模、上下文组织、研判推理、候选方案生成、流程触发、结果解释和经验回流。

## 2. 雷达智能体关键能力体系

雷达智能体的研究路线应围绕关键能力体系展开，而不是按照软件模块、接口表或部署单元重复描述。关键能力体系体现的是建设雷达智能体必须形成的核心技术能力，以及这些能力在雷达任务闭环中的作用。

从智能体工程实践看，雷达智能体需要把上下文工程、Agent Harness 运行底座、领域 Skills 能力包、工具调用治理、工作流门控、结构化输出、观测评估和记忆治理等方法融入能力设计。上下文工程用于保证模型在正确任务对象、正确证据范围和正确工程约束下推理；Agent Harness 用于承载任务生命周期、状态流转、工具编排、安全门控和运行观测；雷达 Skills 能力包用于沉淀目标研判、参数推荐、异常解释和复盘生成等领域能力；工具与工作流治理用于将开放式推理纳入可控执行链路；评估和追溯机制用于检验智能体输出是否可信、可解释、可复盘。

### 2.1 任务理解与运行组织能力

任务理解与运行组织能力是雷达智能体进入工程系统的起点，重点解决多源触发信息如何转化为可管理、可调度、可追溯的智能体任务。雷达智能体面对的输入不仅包括操作员自然语言指令，还包括显控界面的目标选择、区域框选、告警卡片、参数面板状态，以及信息处理感知结果事件、雷达状态告警和控制执行回传。研究路线需要将这些输入统一抽象为任务对象，使智能体后续推理始终围绕明确的任务编号、目标对象、时间窗口和运行状态展开。

该能力的设计重点是建立结构化任务建模方法，并由 Agent Harness 承载任务生命周期管理。任务对象应至少包含任务类型、触发来源、目标或区域引用、时间窗口、输入证据、风险等级、上下文版本、执行状态和追溯标识。Harness 需要维护任务创建、任务分级、上下文装载、能力选择、工具调用、工作流触发、状态同步、结果发布和任务归档等运行状态，使智能体任务能够被调度、监控和回放。对于连续交互任务，还需要维护多轮上下文中的指代关系，例如“这个目标”“沿用上次方案”“继续刚才的研判”等表达需要映射到确定的目标编号、任务编号和历史方案。对于事件触发任务，需要将感知事件、状态告警和显控任务纳入同一任务运行框架，避免事件处理、人工交互和智能推理之间形成割裂链路。

落地时，任务理解能力应形成任务受理、任务建模、任务分级、任务状态维护和任务归档等基本机制。系统接收输入后先完成意图解析和对象绑定，再根据任务类型、实时性要求和风险等级选择处理路径。只读研判任务进入动态推理链路，参数建议任务进入候选方案生成链路，涉及控制请求或参数变更的任务进入受控流程链路。该能力的直接成果包括任务对象规范、任务状态流转规则、Harness 运行状态定义、典型任务样例、连续对话对象绑定样例和任务追溯编号规则。

### 2.2 雷达态势认知与证据组织能力

雷达态势认知与证据组织能力是智能体形成可靠研判的基础，重点解决推理依据如何从分散数据转化为可解释的任务工作空间。雷达智能体不能只依赖自然语言输入或模型内部知识，需要面向当前目标、当前区域和当前时间窗口，组织信息处理感知结果、雷达工作状态、环境条件、历史案例、专业知识、时序指标和图关系信息。该能力的核心是把多源信息变成可引用、可校验、可复盘的证据集合。

该能力本质上是面向雷达任务的上下文工程。上下文工程不只是扩大模型输入长度，而是根据任务目标、风险等级和输出要求选择合适的上下文范围，明确哪些信息进入当前推理、哪些信息作为证据引用、哪些信息只作为补充背景。感知结果侧应包含目标检测、识别、跟踪、定位、置信度、异常评分和模型版本等信息；雷达状态侧应包含当前工作模式、波形参数、波束资源占用、设备健康状态、告警状态和参数版本等信息；知识经验侧应包含专业知识、历史案例、参数策略、操作规程和复盘结论等信息。系统在组织证据时，需要区分观测事实、感知模型判断、规则约束、历史经验、推理结论和人工反馈，避免把不同来源、不同可信度的信息混同使用。

落地时，态势认知能力应围绕目标研判和参数推荐两类核心场景建立证据模板和上下文装载策略。目标研判场景需要形成目标类别、行为特征、航迹变化、异常程度、威胁等级、证据缺口和处置建议等输出依据；参数推荐场景需要形成任务目标、目标状态、环境约束、资源约束、当前参数、历史效果和风险条件等输入依据。系统输出结论时，应同步给出证据引用、数据时间、来源版本、置信度和适用边界，使操作员能够理解研判来源，使后续复盘能够还原当时的判断条件。该能力的直接成果包括任务工作空间组织方法、上下文装载策略、证据项定义、证据链输出样例、数据质量标识规则和目标研判证据模板。

### 2.3 智能推理与参数推荐能力

智能推理与参数推荐能力是雷达智能体体现智能价值的核心，重点解决大语言模型如何在雷达专业约束下生成可审查、可比较、可进入流程的业务结果。该能力不是让模型脱离工程链路直接给出控制动作，而是利用大模型的语义理解、知识整合和任务规划能力，结合雷达 Skills 能力包、任务上下文、工具结果和工作流约束，生成面向目标研判、智能调度和波形参数设置的结构化输出。

在目标研判方面，智能推理应围绕当前环境和目标状态开展多证据综合分析，输出目标解释、异常原因、跟踪质量评估、证据缺口、风险提示和后续处置建议。系统需要把感知结果中的类别、航迹、置信度、异常评分与历史案例、环境信息和专业知识结合起来，形成“事实依据-推理判断-不确定性说明-建议动作”的结果结构。雷达 Skills 能力包应沉淀可复用的任务提示、领域知识引用规则、工具选择规则和结构化输出格式，使不同场景下的推理过程具有一致的工程约束。对于证据不足、感知结果冲突或状态异常的情况，系统应降低结论确定性，提出补充观测、补充查询或人工复核要求。

在智能调度和波形参数设置方面，参数推荐应面向任务目标、目标状态、雷达资源、环境条件和当前工作模式生成候选方案。候选方案应包含推荐参数、推荐依据、适用条件、预期效果、风险点、验证方式和回滚条件。对于波形参数、检测门限、跟踪参数和波束资源等内容，系统应避免给出孤立数值，而应说明参数调整与目标特征、环境约束、资源占用和历史执行效果之间的关系。该能力还需要建立结构化输出和结果评估机制，对事实性、证据一致性、参数边界、风险表达和可执行性进行检查。该能力的直接成果包括雷达 Skills 能力包设计、研判结果结构模板、候选参数方案模板、参数推荐约束清单、风险说明模板、输出质量检查规则和典型场景推理样例。

### 2.4 工具调用与受控流程能力

工具调用与受控流程能力是雷达智能体实现工程落地的关键，重点解决开放式智能推理如何与确定性工具、标准流程和高风险控制链路协同。雷达智能体需要调用目标分析、数据查询、状态解释、参数仿真、报告生成和控制请求适配等专业能力，但工具调用不能变成无约束的自由执行。研究路线需要将工具划分为只读分析类、仿真评估类、流程触发类和控制请求适配类，并根据任务风险和标准化程度选择不同执行路径。

对于只读研判任务，系统可采用推理-行动-观察式工具调用方式，由认知推理过程根据证据缺口动态选择查询、分析和解释工具，观察工具返回结果后继续修正研判。对于参数推荐任务，系统应调用参数仿真、历史效果查询、资源约束检查等工具，为候选方案提供验证依据。对于参数设置、工作模式切换、控制请求适配和异常回滚等高风险任务，系统应进入受控工作流，按照模板完成参数边界校验、仿真评估、人工确认、检查点记录、请求适配、状态反馈和异常处置。该过程体现智能体落地的关键实践：模型负责分析、规划和候选方案生成，工具和工作流负责确定性计算、状态检查、流程约束和工程动作衔接。

落地时，该能力需要形成工具调用规范、工作流模板和风险分级机制。工具调用结果应以标准化结构返回，包括执行状态、输入参数、输出结果、证据编号、异常信息、耗时信息和追溯引用。工作流执行过程应记录节点状态、审批动作、检查点、回滚条件和执行反馈。控制请求适配应作为受控流程中的专业工具或适配环节，承接经审批的候选方案并与控制执行链路衔接。Agent Harness 应对工具权限、参数边界、调用超时、异常回退和审计记录进行统一治理。该能力的直接成果包括工具能力清单、工具调用结果规范、受控工作流样例、风险分级规则、工具治理规则和参数调整闭环演示流程。

### 2.5 记忆沉淀与持续演进能力

记忆沉淀与持续演进能力是雷达智能体从单次任务处理走向长期可用系统的基础，重点解决任务过程、操作经验和执行效果如何沉淀为可评估、可复用、可治理的经验资产。雷达智能体需要的不只是历史记录归档，而是能够区分当前任务上下文、近期任务链和长期知识经验的分层记忆机制，使系统既能支撑当前任务连续推理，又能在后续相似任务中受控引用历史经验。

短期记忆面向当前任务，保存当前输入、上下文快照、工具调用结果、工作流状态、中间推理依据和待确认事项，支撑多轮交互和任务内连续推理。中期记忆面向近期任务链，维护目标、区域、参数方案、状态变化、操作反馈和任务结果之间的关联，支撑连续跟踪、方案对比和跨任务复盘。长期记忆面向知识、案例、策略和经验资产，保存经过评估的成功经验、失败案例、典型参数策略、处置流程和复盘结论，为后续任务提供检索增强和经验参考。

落地时，记忆沉淀应与追溯记录、效果评估和人工反馈结合，形成“记录-评估-抽取-确认-复用”的闭环。任务结束后，系统应从追溯记录中提取候选经验，并结合执行效果、操作员反馈、参数变化前后状态和任务结果进行评估。通过评估的经验进入长期资产库，未通过评估的记录保留为历史任务材料，不直接作为推荐依据。后续任务引用历史经验时，应返回经验来源、适用条件、冲突提示和追溯引用，保证经验复用可解释、可审查、可回退。该能力还需要建立智能体评估样例集，围绕目标研判准确性、证据引用完整性、参数推荐合理性、流程合规性和经验复用有效性开展持续评测。该能力的直接成果包括三阶记忆对象定义、经验候选抽取规则、复盘评估样例、长期经验资产样例、经验受控复用机制和智能体评估样例集。

上述五类能力共同构成雷达智能体的研究主线：任务理解与运行组织解决“任务如何进入智能体”；态势认知与证据组织解决“推理依据从哪里来”；智能推理与参数推荐解决“智能体生成什么结果”；工具调用与受控流程解决“结果如何进入工程链路”；记忆沉淀与持续演进解决“任务经验如何反馈到后续能力提升”。通过这一能力体系，雷达智能体能够从单次智能问答转向面向雷达任务的持续运行系统。

## 3. 雷达智能体技术路线与阶段成果

雷达智能体技术路线以任务闭环为主线展开，整体流程为：任务触发、Agent Harness 任务实例化、上下文工程装载、LLM 与雷达 Skills 推理、工具或工作流受控调用、结构化结果发布、追溯记录、评估反馈和记忆回流。该路线将大模型推理能力纳入雷达工程约束，使智能体既具备开放式研判和自然语言交互能力，又能够通过 Harness、工作流、门控校验、追溯记录、评估机制和记忆治理保持工程可控。

在任务触发阶段，显控指令、感知结果事件和雷达状态告警进入雷达智能体，由任务运行机制创建任务实例，绑定任务编号、目标或区域引用、任务类型、风险等级和追溯标识。在上下文装载阶段，系统围绕任务实例加载当前感知结果、雷达状态、历史任务、相似案例、专业知识和数据质量信息，形成任务工作空间。在智能推理阶段，LLM 推理引擎结合雷达 Skills 能力包完成意图理解、任务建模、执行路径选择和结果生成。对于只读研判和推荐生成任务，系统通过工具调用补充证据、分析状态和生成结果；对于参数设置和控制相关任务，系统将候选方案提交受控工作流，完成参数边界校验、仿真评估、人工确认、控制请求适配和状态反馈。任务结束后，系统将输入、上下文版本、工具结果、工作流节点、审批动作、输出结论、操作员反馈和执行效果写入追溯记录，并按照短期、中期、长期记忆机制进行归档、评估和回流。

第一阶段形成基础能力验证成果。该阶段重点完成任务建模、上下文组织、三阶记忆和追溯记录框架，验证雷达智能体能否稳定接收显控任务、感知事件和状态告警，能否形成结构化任务对象，能否围绕目标、区域、时间窗口和风险等级组织上下文，能否记录任务过程并形成可复盘的追溯链。阶段成果包括任务对象样例、上下文组织样例、三阶记忆框架、追溯记录样例和基础任务闭环原型。

第二阶段形成典型场景验证成果。该阶段重点围绕当前环境目标研判、智能调度建议和波形参数推荐开展验证，检验智能体能否综合感知结果、雷达状态、环境条件、历史案例和专业知识生成可解释结果。目标研判场景应形成目标类别、行为特征、异常程度、证据链和处置建议；调度与波形参数推荐场景应形成候选参数方案、推荐依据、适用条件、预期效果、风险说明和回滚条件。阶段成果包括典型任务样例库、目标研判演示样例、参数推荐演示样例、证据链输出模板和显控展示样例。

第三阶段形成闭环演进验证成果。该阶段重点验证工具调用、工作流受控执行、复盘评估和经验沉淀能力，推动雷达智能体从单次任务处理走向持续运行和持续优化。系统应能够将参数仿真、人工确认、控制请求适配、执行状态反馈和异常回滚纳入受控流程；能够基于任务追溯记录和执行效果生成复盘结论；能够将经过评估的任务经验沉淀为长期经验资产，并在后续相似任务中受控引用。阶段成果包括受控工作流样例、参数调整闭环演示、复盘评估报告样例、经验资产样例和持续优化验证结果。

通过上述三个阶段，雷达智能体研究路线从基础任务闭环入手，逐步扩展到典型雷达智能应用场景，再进入受控执行和经验演进闭环。最终形成面向雷达内部任务的智能推理、参数推荐、流程协同和持续学习能力，为后续软件需求规格说明、概要设计、接口控制文档和原型系统建设提供研究基础。
