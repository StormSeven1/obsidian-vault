# 节点一三阶分层记忆架构关键技术研究报告提纲

## 1. 报告定位

本报告作为节点一技术成果，面向雷达智能体项目的关键技术攻关验收，重点说明三阶分层记忆架构如何支撑雷达智能体从单次问答式推理升级为连续任务运行、经验沉淀和受控回流的工程能力。

报告应回答五个问题：

1. 为什么雷达智能体必须采用三阶分层记忆架构。
2. 短期记忆、中期记忆、长期记忆分别解决什么问题。
3. 三层记忆如何通过任务实例、任务上下文和追溯记录贯通。
4. 三阶记忆如何与 Agent Harness、Skills、工具执行、工作流引擎和数据库底座协同。
5. 节点一已经形成哪些设计成果，后续如何通过工程样例验证。

## 2. 建议章节

### 2.1 技术背景与问题定义

说明雷达智能体面临的连续任务问题，包括目标连续跟踪、参数多次调整、模型置信度变化、操作员多轮交互、历史经验复用和控制动作可追溯等。

重点突出：如果没有分层记忆，智能体只能处理孤立任务，难以支撑目标研判、波形参数设置、异常归因、流程执行和经验沉淀闭环。

### 2.2 三阶分层记忆总体架构

说明三阶记忆的总体划分：

| 记忆层级 | 作用对象 | 核心内容 | 主要用途 |
| --- | --- | --- | --- |
| 短期记忆 | 当前任务 | 任务上下文、当前证据、工具结果、候选方案、工作流检查点 | 支撑当前目标研判、参数调度和流程执行 |
| 中期记忆 | 近期任务链 | 目标研判链、区域任务链、参数调整链、异常事件链 | 支撑多轮指代、连续任务和近期状态回看 |
| 长期记忆 | 经验资产 | 雷达知识、历史案例、参数策略、异常模式、经验资产 | 支撑相似案例检索、策略推荐和经验复用 |

总体架构建议表述为：

```text
当前任务上下文
  → 任务追溯记录
  → 目标/区域/参数/异常任务链
  → 候选经验片段
  → 长期经验资产
  → 相似案例、策略推荐、工具匹配和工作流优化回流
```

### 2.3 短期记忆关键技术

重点说明短期记忆如何支撑当前任务稳定执行。

建议覆盖：

1. 任务上下文模板。
2. 上下文版本管理。
3. 上下文投影机制。
4. 工具结果缓存与证据编号。
5. 工作流检查点。
6. 证据一致性校验。

节点一成果应体现：

1. 已明确目标研判上下文字段。
2. 已明确参数调度上下文字段。
3. 已明确工具结果如何进入证据链。
4. 已明确任务完成后如何生成追溯记录。

### 2.4 中期记忆关键技术

重点说明中期记忆如何解决近期连续任务协同。

建议覆盖四类任务链：

1. 目标研判链：记录目标检测、识别、航迹、异常评分、研判结论和证据变化。
2. 区域任务链：记录区域范围、关注目标、环境状态、参数方案和执行效果。
3. 参数调整链：记录调整前状态、候选方案、审批结果、下发时间、效果指标和回滚情况。
4. 异常事件链：记录发现、研判、补证、确认、处置、观察和复盘状态。

节点一成果应体现：

1. 已明确任务链索引字段。
2. 已明确“刚才”“上次”“继续前一次”等多轮指代的对象绑定方式。
3. 已明确任务链与短期任务上下文、长期经验条目的引用关系。

### 2.5 长期记忆关键技术

重点说明长期记忆如何支撑知识和经验复用。

建议采用“四库”结构：

1. 知识库：雷达技术文档、设备手册、操作规程、波形原理、参数约束。
2. 案例库：目标研判案例、参数调整案例、干扰处置案例、跟踪异常案例、回滚恢复案例。
3. 策略库：波形参数策略、资源调度策略、门限调整策略、跟踪参数优化策略。
4. 经验库：经过复盘、评估、发布的经验资产。

节点一成果应体现：

1. 已明确长期记忆资产分类。
2. 已明确相似案例检索的标签维度。
3. 已明确经验资产的来源任务、证据引用、质量状态和版本状态。

### 2.6 统一追溯记录机制

重点说明追溯记录是三层记忆贯通的枢纽。

追溯记录建议至少包括：

1. 任务标识、触发来源、任务类型、目标/区域引用。
2. 显控上下文、模型输出、雷达状态、上下文版本。
3. Skill 选择、工具调用、工作流节点、审批确认。
4. 研判结果、候选参数方案、执行结果、效果指标。
5. 操作员反馈、异常记录、经验抽取状态。

报告中要强调：追溯记录不是普通日志，而是支撑审计、回放、复盘和经验抽取的核心数据对象。

### 2.7 多形态存储与数据底座

说明三阶记忆不是单一数据库，而是多形态数据组合。

建议表述：

| 数据形态 | 承载内容 | 服务能力 |
| --- | --- | --- |
| 关系型数据 | 任务、参数、审批、工具调用、工作流实例、追溯记录 | 结构化查询和审计 |
| 向量数据 | 文档知识、案例摘要、经验片段 | 语义检索和相似案例召回 |
| 图数据 | 目标、环境、参数、设备、异常、策略关系 | 关系推理和异常归因 |
| 时序数据 | 雷达状态、航迹、模型置信度、参数效果、设备健康 | 趋势分析、效果监控和回滚判断 |

### 2.8 经验资产治理与安全回流

说明经验不能直接进入长期记忆并影响后续控制动作，必须经过治理流程。

建议流程：

```text
任务追溯记录
  → 候选经验片段
  → 情景摘要与模式标签
  → 效果指标与失败原因分析
  → 专家复核或规则校验
  → 影子运行验证
  → 已发布经验资产
  → 受控回流
```

回流形式限定为：

1. 相似案例检索依据。
2. 候选参数策略。
3. 工具匹配信号。
4. 工作流模板优化建议。

报告中要明确：经验回流不绕过参数边界校验、工作流审批和人工确认。

### 2.9 与系统模块协同关系

说明三阶记忆与各模块的协同：

| 模块 | 与三阶记忆的关系 |
| --- | --- |
| Agent Harness | 创建任务实例，装载上下文，触发记忆读写，生成追溯记录 |
| 认知核心 | 决定记忆检索、上下文投影、证据使用和结果写入 |
| Skills | 声明记忆依赖、读写规则、输出格式和经验候选项 |
| 工具执行模块 | 返回标准化工具结果，生成证据编号并写入追溯记录 |
| 工作流引擎 | 写入节点状态、检查点、审批记录、回滚点和执行结果 |
| 数据库管理模块 | 承载关系、向量、图、时序和追溯数据 |

### 2.10 节点一成果与后续验证

节点一建议交付成果：

1. 三阶分层记忆总体架构设计。
2. 短期任务上下文对象模型。
3. 中期任务链对象模型。
4. 长期经验资产分类与索引设计。
5. 统一追溯记录字段设计。
6. 记忆读写接口与模块协同机制。
7. 三类工程验证样例设计。

后续验证样例：

1. 目标研判样例：验证短期任务上下文、证据链和目标研判链沉淀。
2. 波形参数调整样例：验证参数调整链、效果指标记录、工作流检查点和回滚条件。
3. 复盘学习样例：验证追溯记录到候选经验，再到长期经验资产的治理流程。

## 3. 节点一报告建议结论

节点一报告可形成以下结论：

1. 三阶分层记忆架构是雷达智能体实现连续任务协同和经验演进的关键技术。
2. 短期记忆解决当前任务上下文稳定执行问题，中期记忆解决近期目标、区域、参数和异常事件连续协同问题，长期记忆解决知识经验沉淀复用问题。
3. 统一追溯记录是三层记忆贯通和后续审计、复盘、经验抽取的核心机制。
4. 三阶记忆必须与 Agent Harness、Skills、工具、工作流和数据库底座协同实现，而不是独立存储模块。
5. 节点一已完成架构、对象、流程和接口层面的关键技术设计，后续将通过目标研判、波形参数调整和复盘学习样例进行工程验证。
