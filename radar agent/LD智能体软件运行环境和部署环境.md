# LD 智能体软件运行环境和部署环境

## 1. 软件运行环境

|环境项|要求|
|---|---|
|硬件平台|国产 910B AI 服务器|
|操作系统|64 位 Linux 操作系统|
|后端运行时|Python 3.x，配套 pip/uv 等依赖管理工具|
|后端服务框架|支持 HTTP/gRPC/WebSocket 等接口服务能力，可根据工程选型采用 FastAPI、Flask 或等效 Python 服务框架|
|前端运行时|Node.js LTS，配套 npm/pnpm 等包管理工具|
|前端框架|Next.js，用于智能体交互页面、任务状态展示、研判结果展示和工作流状态展示|
|AI 推理环境|适配 910B 的 NPU 驱动、CANN、MindIE 或等效大模型推理框架|
|模型运行资源|支持本地大语言模型或模型服务网关调用，具备模型加载、推理队列、超时控制和资源监测能力|
|数据存储|可访问关系数据库、向量检索/语义检索服务、文件或对象存储|
|通信接口|支持 HTTP/gRPC、WebSocket、消息接口、数据库接口和文件/对象存储接口|
|运维支撑|具备日志记录、健康检查、资源监控、配置加载和异常告警能力|

## 2. 部署环境

|部署项|要求|
|---|---|
|服务器|国产 910B AI 服务器，可根据任务规模配置单机或多机部署|
|操作系统|64 位 Linux，系统运行库需与 Python、Node.js 和 910B 推理框架兼容|
|后端部署|部署 Python 后端服务，提供任务提交、智能对话、任务状态、智能研判、工作流触发和工具调用等接口|
|前端部署|部署 Next.js 前端服务，可通过 Node.js 运行服务或 Nginx 等反向代理发布|
|推理服务部署|部署适配 910B 的大模型推理服务，完成模型权重、Tokenizer、推理参数、NPU 资源和日志目录配置|
|数据库部署|部署或接入关系数据库，用于保存任务、会话、状态、审批、追溯和配置类数据|
|记忆与知识存储|部署或接入向量检索/语义检索服务、知识库和文件/对象存储，用于支撑三阶分层记忆、案例检索和经验沉淀|
|反向代理|可部署 Nginx 或等效组件，用于 API 转发、HTTPS 终结、WebSocket 升级和静态资源服务|
|服务管理|支持 systemd、Docker 或等效方式进行服务启动、停止、重启和开机自启动管理|
|网络要求|部署节点需与显控分系统、信息处理感知子系统、阵面状态服务、记录回放子系统、数据知识底座和权限审计服务保持网络连通|
|安全配置|数据库连接、模型路径、接口地址、访问令牌和密钥应通过配置文件或环境变量注入|
|监控日志|采集 CPU、内存、磁盘、网络、NPU 利用率、NPU 显存、推理耗时、任务状态和接口调用异常等运行信息|
