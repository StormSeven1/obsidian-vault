# LD 智能体软件运行环境和部署环境

## 1. 编制范围

本文面向 LD 智能体子系统的软件运行环境和部署环境进行说明，重点描述国产 910B AI 服务器、Linux 操作系统、Python 后端、Next.js 前端、大模型推理服务、数据与记忆存储、接口通信、运维监控和安全配置等工程部署要求。本文可作为系统技术方案中 LD 智能体子系统运行部署内容的独立附件使用。

LD 智能体子系统是多功能智能 LD 系统中的智能决策与人机协同子系统，主要承担任务理解、智能研判、参数建议、工具调用、工作流编排、三阶分层记忆、追溯记录和显控协同等能力。其运行环境应在国产化软硬件平台上支撑在线任务、准实时研判、受控流程和离线经验演进稳定运行。

## 2. 软件运行环境

LD 智能体子系统软件运行环境面向国产化 AI 计算平台建设，运行硬件采用国产 910B AI 服务器，操作系统采用 64 位 Linux 系统。子系统运行环境由智能体后端服务、模型推理运行环境、前端交互服务、数据与记忆存储环境、接口通信与运维支撑环境组成。

### 2.1 硬件与操作系统环境

运行服务器采用国产 910B AI 服务器，提供 CPU、内存、磁盘、网络和昇腾 NPU 计算资源。服务器应具备承载大模型推理、智能体后端服务、Web 前端服务、数据库/记忆存储访问、日志监控和接口通信的基础能力。

操作系统采用 64 位 Linux 系统，系统环境应与 Python 运行时、Node.js 运行时、CANN/MindIE 推理框架、数据库客户端、对象存储客户端和监控组件兼容。系统应提供稳定的网络配置、时间同步、磁盘挂载、服务进程管理、日志目录、权限控制和基础安全加固能力。

### 2.2 Python 后端运行环境

智能体后端采用 Python 技术栈实现，主要承载任务接入、上下文装载、能力包调度、LLM 推理编排、工具调用适配、工作流触发、安全门控、任务状态管理和追溯记录等服务能力。

后端运行环境应提供 Python 3 运行时、虚拟环境或依赖管理工具、HTTP/gRPC/WebSocket 服务框架、消息或事件订阅能力、数据库客户端、对象存储客户端、日志采集组件和配置加载能力。后端服务通过配置文件或环境变量接入显控分系统、信息处理感知子系统、阵面状态服务、数据知识底座、记录回放子系统、任务与实体管理服务以及权限审计服务。

后端配置应避免将现场地址、密钥、模型路径、接口参数和鉴权令牌固化在程序代码中。不同部署环境应通过受控配置区分开发、联调、试验和生产运行参数。

### 2.3 大模型推理运行环境

模型推理运行环境部署在 910B AI 服务器的昇腾计算资源之上，配套 NPU 驱动、CANN、MindIE 或其他适配 910B 的大模型推理框架，支撑本地大语言模型、工具调用规划模型或外部模型服务网关的统一接入。

推理服务应支持模型加载、实例管理、请求队列、上下文长度控制、推理超时控制、资源占用观测、异常反馈和日志记录。对于准实时智能研判、报告生成、经验抽取等任务，可通过独立推理服务或独立队列运行；对于控制类任务，推理输出只形成研判结论、参数候选方案和工作流触发建议，后续仍需进入安全门控、流程审批和人工确认链路。

推理运行环境应对模型实例数量、并发请求数、上下文长度、单请求超时、NPU 显存、主机内存和日志目录设置资源边界。模型加载失败、资源不足、推理超时或服务异常时，应向后端返回明确状态，由后端执行降级、重试、人工提示或任务失败记录。

### 2.4 Next.js 前端运行环境

前端采用 Next.js 技术栈实现，运行环境包括 Node.js LTS、包管理工具、构建工具链和 Web 运行服务。前端主要用于 LD 智能体相关的人机交互页面、自然语言对话、任务状态查看、研判结果展示、证据链展示、工作流审批状态展示和处置方案确认等界面能力。

前端通过统一 API 地址访问 Python 后端服务，并根据业务需要接入 WebSocket 或流式接口接收任务状态、智能研判结果、工具执行片段和工作流节点状态。生产运行时可采用 Next.js 服务方式运行，也可在构建后通过 Nginx 等反向代理统一发布。

### 2.5 数据与记忆存储环境

数据与记忆存储环境用于保存任务实例、短期上下文、中期任务链、长期知识与经验索引、工具执行结果、工作流状态、审批记录、操作员反馈和追溯记录。

部署环境应提供可达的关系数据库、向量检索或语义检索服务、文件/对象存储、日志存储和备份恢复能力。具体数据库产品、向量库产品、表结构、索引策略和存储容量在详细设计阶段结合数据规模、查询模式和国产化适配要求确定。

### 2.6 接口通信与运维支撑环境

接口通信环境应支持 HTTP/gRPC、WebSocket、消息接口、数据库接口和文件/对象存储接口。子系统需与智能 LD 显控分系统、信息处理感知子系统、阵面控制分系统、记录回放子系统、任务与实体管理服务、数据知识底座、权限审计系统以及外部智能服务保持网络连通。

运维支撑环境应提供服务启动、停止、重启、健康检查、日志轮转、资源监控、配置管理和异常告警能力。运行监控应覆盖 CPU、内存、磁盘、网络、NPU 利用率、NPU 显存、推理队列长度、请求耗时、任务成功率、工作流异常和接口调用失败等指标。

## 3. 部署环境

LD 智能体子系统部署在国产 910B 服务器和 Linux 操作系统环境中，部署形态可采用裸机服务、容器化服务或二者结合的方式。典型部署由 Python 后端服务、LLM 推理服务、Next.js 前端服务、数据库/记忆存储服务、对象存储服务、日志监控服务和反向代理服务组成。

各服务之间通过 HTTP/gRPC、WebSocket、消息接口、数据库接口和文件/对象存储接口协同运行，形成面向显控交互、智能研判、工作流执行和经验回流的完整运行链路。

### 3.1 后端服务部署

后端服务宜按功能拆分为 API 服务、任务运行服务、工作流服务、工具执行服务、记忆检索服务和异步后台任务服务。

API 服务负责对外提供任务提交、状态查询、智能对话、研判结果获取和工作流触发等接口；任务运行服务负责任务生命周期管理、上下文组织和能力包调度；工具执行服务负责封装数据查询、感知结果解释、参数仿真、控制请求适配和报告生成等受控工具；记忆检索服务负责三阶分层记忆的读写、召回、归档和经验候选管理；异步后台任务服务负责报告生成、经验抽取、索引构建和离线复盘等非实时任务。

实际部署时可根据服务器资源、并发规模和可靠性要求合并或拆分进程。联调阶段可采用较简化的一体化部署；长期运行或外场部署阶段应根据服务压力和故障隔离要求进行进程或容器拆分。

### 3.2 推理服务部署

推理服务应与 Python 后端服务解耦部署，通过本地网络接口或进程间接口提供模型调用能力。910B 服务器应完成 NPU 驱动、CANN、推理框架、模型权重、模型配置、Tokenizer、推理参数和运行日志目录配置。

推理服务部署时应明确模型版本、模型路径、启动参数、设备绑定、并发限制、上下文窗口、超时阈值、日志级别和健康检查接口。推理服务异常、模型加载失败、资源不足或响应超时时，应向后端返回明确状态，便于后端进行降级处理、任务状态更新和追溯记录写入。

### 3.3 前端服务部署

Next.js 前端服务可与后端服务部署在同一台 910B 服务器，也可部署在独立 Web 节点。生产部署时，前端构建产物通过 Node.js 运行服务或 Nginx 等反向代理对外发布。

反向代理统一处理 HTTPS 终结、静态资源缓存、API 转发、WebSocket 升级、跨域控制和长连接超时策略。前端运行配置包括后端 API 地址、WebSocket 地址、登录鉴权地址、地图/资源地址、会话超时、心跳周期和页面功能开关，应通过部署配置注入。

### 3.4 数据与存储服务部署

数据与存储服务包括关系数据库、向量检索或语义检索服务、文件/对象存储、日志存储和备份恢复服务。关系数据库用于保存任务、会话、状态、审批、追溯和配置类结构化数据；语义检索服务用于支撑知识、案例、经验和记忆召回；文件/对象存储用于保存报告、附件、模型引用、回放索引和导出文件。

数据服务应具备访问控制、备份恢复、容量监控和日志审计能力。生产环境应对数据库连接、存储桶、访问密钥、数据保留周期和备份策略进行统一配置管理。

### 3.5 运维监控与安全配置

部署环境应具备服务状态监控、接口调用监控、推理资源监控、任务状态监控、日志检索和异常告警能力。关键指标包括服务存活状态、请求成功率、接口耗时、推理队列长度、NPU 利用率、NPU 显存占用、内存占用、磁盘容量、任务失败率和工作流异常数量。

权限密钥、数据库连接、模型路径、服务令牌和外部接口地址应纳入受控配置管理。生产环境应关闭调试模式，限制管理接口访问范围，控制跨域来源和上传大小，保留任务、工具、工作流、控制请求和人工确认等关键操作的审计记录。

### 3.6 部署拓扑建议

部署拓扑应按照在线任务和离线任务隔离、推理服务和业务服务解耦、前端访问和后端接口分层、关键配置可回滚的原则组织。

联调和试验环境可采用单机一体化部署，即在同一台 910B 服务器上部署 Python 后端服务、推理服务、Next.js 前端服务和必要的数据访问组件，便于快速验证任务闭环、接口连通性和显控交互流程。

长期运行或外场部署环境应根据并发量和可靠性要求将推理服务、后端服务、前端服务、数据库和对象存储进行资源隔离。推理服务优先绑定 910B NPU 资源，后端服务负责业务编排和接口管理，前端服务通过反向代理统一发布，数据库和对象存储提供独立备份与容量管理，并预留模型版本切换、配置回退、日志导出和故障诊断通道。

## 4. 环境要素汇总

|类别|环境要素|说明|
|---|---|---|
|硬件平台|国产 910B AI 服务器|承载智能体后端、大模型推理、前端服务和运行支撑组件|
|操作系统|64 位 Linux|与 Python、Node.js、CANN/MindIE、数据库客户端和监控组件兼容|
|后端技术栈|Python|承载任务接入、推理编排、工具调用、工作流、安全门控和追溯记录|
|前端技术栈|Next.js、Node.js LTS|承载智能体交互页面、任务状态、研判结果和审批确认展示|
|推理环境|NPU 驱动、CANN、MindIE 或等效 910B 推理框架|承载本地大模型或模型服务网关调用|
|通信接口|HTTP/gRPC、WebSocket、消息接口、数据库接口、文件/对象存储接口|支撑显控交互、状态推送、任务调用、数据访问和追溯记录|
|数据存储|关系数据库、语义检索/向量检索、文件/对象存储|保存任务、记忆、知识、经验、追溯、报告和导出文件|
|运维监控|日志、指标、健康检查、告警、审计|覆盖服务状态、接口状态、推理资源、任务状态和关键操作|
