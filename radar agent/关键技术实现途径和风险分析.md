## 关键问题与技术重要性

三阶分层记忆架构解决的不是普通数据存储问题，而是主流 Agent 架构在工程落地中必须处理的上下文管理、知识接地、行动闭环和经验演进问题。ReAct 类智能体强调“推理-行动-观察”的交替运行，关键难点在于每一次工具调用、环境观察和中间结论都会改变任务状态，系统必须持续维护当前计划、已用证据、待补证据和下一步动作；RAG 类架构强调通过外部知识检索增强模型生成，关键难点在于检索结果需要与任务目标、时间窗口、数据质量和适用条件对齐，避免语义相似但工程条件不同的信息进入推理；长期运行智能体还面临上下文窗口有限与任务历史不断增长之间的矛盾，需要像虚拟上下文管理一样，在当前上下文、近期任务链和长期经验资产之间进行选择、压缩、召回和更新。雷达智能体面向当前环境目标研判、智能雷达调度和波形参数设置，输入同时包含感知结果、雷达状态、显控交互、工具调用、工作流节点、审批反馈和历史任务，任务过程具有多源输入、状态连续、时效敏感、强安全约束等特点，因此必须通过三阶分层记忆把“模型上下文”提升为“可治理的任务认知空间”。

在该任务认知空间中，短期记忆承担当前任务的工作记忆作用，围绕单个任务实例保存任务意图、目标/区域引用、时间窗口、雷达状态摘要、感知证据、工具观察结果、候选参数方案、风险判断和工作流检查点，保证认知核心在一次研判或参数推荐过程中始终基于同一组可追溯事实进行推理。中期记忆承担连续任务状态管理作用，围绕目标研判链、区域任务链、参数调整链和异常事件链记录近期任务的状态迁移、证据变化、参数版本、审批状态、执行反馈和效果指标，支撑“继续刚才目标研判”“对比上次参数调整效果”“跟踪该异常处置进展”等连续交互和准实时研判。长期记忆承担知识与经验资产管理作用，将专业知识、历史案例、参数策略、异常模式和复盘经验按照来源、适用条件、质量状态、版本状态和风险边界进行治理，用于相似案例检索、策略推荐、工具选择和工作流优化。三阶记忆通过任务实例、上下文版本、证据编号和追溯引用，将开放式智能推理过程固化为可检索、可评估、可回放、可审计、可受控回流的工程对象，是雷达智能体形成目标研判、参数建议、受控执行和经验演进闭环的关键技术。

## 实现途径

工程实现应采用“上下文工程 + 检索增强 + 行动观察闭环 + 经验治理”的组合路径。任务运行管理单元在显控请求、感知结果事件或雷达状态告警触发后创建任务实例，绑定任务编号、目标/区域引用、时间窗口、风险等级、上下文版本和追溯编号，形成 Agent 运行的最小工程单元。认知核心围绕任务实例组织推理过程，记忆管理模块负责分层装载和写入：短期记忆提供当前任务上下文投影，控制进入模型上下文的事实、证据、工具结果和输出约束；中期记忆按目标、区域、参数和异常事件维护任务链索引，解决多轮指代、状态延续和近期效果回看；长期记忆以知识库、案例库、策略库和经验库为主要载体，通过适用条件、来源任务、质量状态和版本状态限定复用范围。

数据和流程上，三阶分层记忆应由统一数据接入能力支撑，将记忆设计为多形态数据、检索策略和追溯记录共同构成的工程体系。关系数据承载任务实例、审批记录、工具调用、工作流实例和追溯记录，保证状态一致和审计回放；向量数据承载专业文档、案例摘要和经验片段，支撑语义召回和相似案例检索；图关系数据承载目标、环境、参数、设备、异常和策略之间的关联关系，支撑异常归因和影响链分析；时序数据承载雷达状态、航迹变化、感知置信度、参数效果和设备健康指标，支撑趋势判断、效果评估和回滚判断。记忆读取进入认知核心前，需要携带来源、时间、版本、适用条件和质量标识；工具调用和工作流执行后的观察结果，需要以证据项、节点状态、执行反馈和效果指标形式写入追溯记录，再由复盘机制抽取候选经验，经过评价后沉淀为长期记忆资产。

## 风险分析

三阶分层记忆的核心风险来自主流 Agent 系统普遍面临的上下文污染、检索误召回、经验过度泛化和反馈闭环失控。雷达任务具有强时效、强条件约束和高安全要求，历史案例或参数策略即使语义相似，也可能因目标类型、环境条件、雷达工作模式、设备健康状态、参数边界或执行时机不同而不适用。若短期记忆缺少上下文版本和证据来源标识，中期任务链缺少稳定对象绑定，长期经验缺少质量评价和版本发布控制，系统可能把过期状态、冲突证据、低质量案例或未经验证的经验注入推理过程，导致目标研判可信度下降、参数建议失真或工作流触发条件判断错误。

风险控制应落实到记忆生命周期治理，而不是依赖模型在推理阶段自行甄别。短期记忆需要执行上下文裁剪、证据一致性校验和工具结果状态标注，确保模型只使用与当前任务、目标和时间窗口匹配的信息；中期记忆需要通过任务编号、目标编号、区域编号、参数版本、时间窗口和追溯引用保持任务链对象一致，避免多轮交互和连续任务中发生对象漂移；长期记忆需要建立候选经验生成、效果指标分析、规则校验或专家复核、影子验证、版本发布和适用范围标注机制。经验回流应定位为相似案例检索、候选参数策略、工具匹配信号和工作流优化依据，并继续接受参数边界校验、安全门控、工作流审批和人工确认约束；对于证据不足、数据质量异常、经验冲突或雷达状态异常的场景，系统应降低结论置信度、触发补证流程或转入人工复核。
