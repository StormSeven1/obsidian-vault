# 三阶分层记忆架构关键技术攻关情况

三阶分层记忆架构是面向雷达智能体连续任务运行、可信推理和经验演进的关键支撑技术，重点解决主流 Agent 架构在工程落地中普遍存在的上下文管理、知识接地、行动闭环和经验回流治理问题。该技术不是将任务记录、历史案例和知识文档进行简单存储，而是要在有限模型上下文内，把当前任务状态、近期任务链路和长期经验资产组织成可检索、可追溯、可治理的任务认知空间。其攻关难点主要体现在三个方面：一是 ReAct 类“推理-行动-观察”循环会使任务状态随工具调用、感知观察和中间结论不断变化，雷达场景中的目标航迹、感知置信度、工作模式、波形参数和设备状态又具有明显时变特征，如果缺少稳定的短期任务记忆，智能体容易重复调用工具、遗漏关键观察、混用不同时刻的状态信息，导致研判结论前后不一致或误触发参数调整流程；二是 RAG 类检索增强需要解决知识接地问题，雷达历史案例和参数策略不仅要语义相关，还要与目标类型、环境条件、时间窗口、雷达状态、参数边界和模型版本匹配，如果仅按文本相似度召回，容易把工程条件不同或已经过期的经验引入推理，造成证据链失真、参数建议失配和风险判断偏差；三是长期运行智能体面临上下文窗口有限与任务历史持续增长之间的矛盾，需要在当前上下文、近期任务链和长期经验资产之间进行选择、压缩、召回和更新，如果缺少分层记忆调度和追溯治理，关键审批记录、回滚条件、执行反馈和效果指标可能在任务流转中丢失，影响连续任务理解、复盘分析和经验复用可靠性。

针对上述问题，本项目按照“任务实例贯穿、三层记忆分工、多形态数据支撑、追溯记录闭环、经验受控回流”的思路开展三阶分层记忆架构攻关。首先，由任务运行管理单元在显控请求、感知结果事件或雷达状态告警触发后创建统一任务实例，绑定任务编号、目标/区域引用、时间窗口、风险等级、上下文版本和追溯编号，作为认知推理、工具调用、工作流执行和记忆读写的共同索引。其次，构建短期、中期、长期三层记忆协同机制：短期记忆面向当前任务工作空间，保存任务意图、雷达状态摘要、感知证据、工具观察结果、候选参数方案、风险判断和工作流检查点，保证当前研判和参数推荐基于同一组可追溯事实；中期记忆面向连续任务状态管理，围绕目标研判链、区域任务链、参数调整链和异常事件链记录状态迁移、证据变化、参数版本、审批状态、执行反馈和效果指标，支撑多轮指代、近期效果回看和准实时任务接续；长期记忆面向知识与经验资产治理，将专业知识、历史案例、参数策略、异常模式和复盘经验按照来源任务、适用条件、质量状态、版本状态和风险边界进行组织，用于相似案例检索、策略推荐、工具选择和工作流优化。最后，通过数据库管理与接入模块将关系数据、向量数据、图关系数据、时序数据和文件归档统一接入，形成“检索召回 + 结构化过滤 + 质量标识 + 追溯写入”的实现路径；工具调用和工作流执行后的观察结果以证据项、节点状态、审批记录、执行反馈和效果指标写入追溯记录，再由复盘机制抽取候选经验，经规则校验、专家复核、效果评价和版本发布后进入长期记忆资产。通过该流程，三阶分层记忆架构将大语言模型的开放式推理过程转化为可回放、可审计、可评估和可受控回流的工程闭环，为雷达智能体开展当前环境目标研判、智能调度和波形参数设置提供持续、可靠的记忆支撑。
